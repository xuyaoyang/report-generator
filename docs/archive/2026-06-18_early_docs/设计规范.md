# 检测报告生成软件 — 设计规范

## UI 设计规范

### 配色方案
- **主色调**：淡蓝色 (#B3D9FF / #A0C8F0)
- **主按钮**：蓝色 (#4A90D9)，hover 加深 (#3A7BC8)
- **背景**：白色 (#FFFFFF)，次要背景浅灰 (#F5F7FA)
- **文字**：主文字深灰 (#333333)，次要文字灰色 (#888888)
- **边框**：浅灰 (#D0D0D0)
- **成功/通过**：绿色 (#52C41A)
- **失败/警告**：红色 (#FF4D4F)

### 字体
- 中文：微软雅黑 (Microsoft YaHei)，9pt-12pt
- 等宽数字：Consolas，用于编号/型号

### 布局原则
- 顶部工具栏：功能按钮区
- 左侧：参数预览/微调面板（占 60% 宽度）
- 右侧：产品型号列表 + 材质单图片预览（占 40% 宽度）
- 底部：状态栏

### 窗口
- 默认大小：1200×800
- 最小尺寸：900×600
- 标题：`{产品类型}检验报告生成系统`

## 代码规范

### Python 风格
- 遵循 PEP8
- 类名：PascalCase（如 `WordEngine`）
- 函数/变量：snake_case（如 `fill_bookmark`）
- 常量：UPPER_SNAKE_CASE（如 `DEFAULT_OUTPUT_DIR`）
- 私有方法：前缀 `_`（如 `_parse_sheet1`）

### 文件组织
- 一个类一个文件（除简单数据类外）
- 模块间通过接口交互，不直接访问内部状态
- 配置与代码分离（JSON 配置文件）

### 错误处理
- UI 层用 try/except 包裹用户操作，弹窗提示错误
- 核心引擎抛出明确异常，由调用方处理
- 所有文件操作使用绝对路径

### 注释原则
- 不写无意义的注释（如 `# 导入模块`）
- 仅在业务逻辑不直观处加简短注释说明 WHY
